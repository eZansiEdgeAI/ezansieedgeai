# Key Learnings

Technical insights:
- Repository was a clean reset with only the core EJS system infrastructure in place (scripts, templates, ADR template)
- The `adr-db.py sync` command initializes the `.ejs.db` SQLite database which is gitignored — it must be re-run per environment
- The EJS journey system uses either YAML frontmatter or plain key:value headers — this file uses plain headers to match the template format

Prompting insights:
- A detailed sub-agent prompt with explicit file-by-file instructions produces reliable results
- Providing the architectural context (vision document contents) in the sub-agent prompt ensures content alignment

Tooling insights:
- PyYAML must be installed for full adr-db.py functionality (falls back to flat key:value parser)
- The `.ejs.db` file is gitignored; `adr-db.py sync` must be re-run after cloning
